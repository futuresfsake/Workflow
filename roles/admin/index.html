How to Use the Dashboard

1

Log in to the Admin Portal

Sign in with your administrator credentials. The Dashboard loads automatically as the default landing page.

2

Review the Summary Widgets

Scan the widget cards for Pending Admissions, Active Trainees, Upcoming Schedules, and Certificates Generated to gauge overall workload.

3

Drill Down into a Widget

Click any widget card to jump directly into the related management module — for example, clicking Pending Admissions opens Review Applications.

4

Check Recent Activities

Scroll through the activity feed to catch actions taken by other staff, such as recently approved applications or encoded attendance.

5

Navigate to a Module

Use the left sidebar to move into Admission, Enrollment, Training, Certificate, Reports, or User Management.

Information

Widget values refresh automatically whenever a related record changes elsewhere in the system — no manual refresh is required.

Warning

The Dashboard only summarizes counts. It does not replace reviewing individual records before approving, rejecting, or generating certificates.

Tip

Check the Dashboard at the start of every shift to prioritize which module needs your attention first.

Expected Result

The administrator has a clear, current snapshot of institutional activity and knows exactly which module to open next.

Step-by-Step Instructions

1

Open Certificate Management

Select Certificate Management → Generate Certificate from the left sidebar.

2

Search for the Trainee

Enter the trainee's name, trainee ID, or batch to locate the correct record. Only trainees who have completed their qualification appear as eligible.

Verify Eligibility

  • Training program fully attended
  • Marked as Competent in the qualification assessed

Each certificate control number is unique and permanently tied to the trainee and qualification — it cannot be reused or reassigned.

Warning

Once generated, a certificate's core details (name, qualification, date) cannot be edited. Voiding and regenerating requires supervisor approval.

Tip

Generate certificates in batches right after a program's completion date to keep the queue from piling up.

Expected Result

A certificate record is created with a unique control number and is ready for preview, printing, or download.

Open Reprint Certificate

Select Certificate Management → Reprint Certificate from the left sidebar.

2

Search for the Certificate

Look up the record by trainee name or certificate control number using the search field.

Enter a Reason for Reprint

Select a reason — Lost, Damaged, or Additional Copy Requested — for the audit trail. This is required before the reprint can proceed.

Search for the User

Locate the account by name, email address, or department.

3

Select Disable Account

Click Disable on the user row. A confirmation dialog appears explaining the effect of disabling the account.

4

Confirm the Action

Click Confirm. The account is immediately deactivated and any active sessions are logged out.

5

Re-Enable if Needed

To restore access later, search for the disabled user and select Enable Account.

Information

Disabling an account preserves all of its historical records and activity logs — it only blocks login access.

Warning

Disabling your own account, or the last remaining Administrator account, is not permitted and will be blocked by the system.

Tip

Prefer disabling over deleting for departed staff — it keeps historical records like generated certificates and approvals traceable to the correct user.

Expected Result

The user account is deactivated and can no longer log in, while its historical data remains intact.

Step-by-Step Instructions

1

Open Assign Roles

Select User Management → Assign Roles from the left sidebar.

2

Search for the User

Locate the account whose role needs to be set or changed.

3

Review Available Roles

Choose the role that matches the user's responsibilities:

  • Administrator — full access to all modules
  • Registrar — Admission, Enrollment, and Certificate modules
  • Trainer — assigned batches, trainee lists, and schedules
  • Assessor — assessment and competency-related records
4

Apply the Role

Select the role from the dropdown and click Apply.

5

Verify Access

The change takes effect immediately. If the user is currently logged in, the new permissions apply on their next action or login.

Information

A user can hold only one role at a time. Assigning a new role replaces the previous one.

Warning

Removing the Administrator role from the last remaining Administrator account is not permitted, to prevent the system from being locked out.

Tip

Review role assignments periodically, especially after staff transfers, to make sure access still matches current responsibilities.

Expected Result

The user's account reflects the new role, with access limited to the modules and actions that role permits.
