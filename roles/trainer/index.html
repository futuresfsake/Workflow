Feature: Class & Student List
Class & Student List Overview
1
Open Trainer Dashboard
Navigate directly to your authorized Instructor workspace from the master SkillTrack deployment directory.
2
Access Assigned Qualifications List
Locate the "Assigned Qualifications" operational card block view containing your active teaching assignments.
3
Select Target Training Batch Allocation
Locate your active structural class block (e.g., CSS-NC2-2026A) and select the corresponding item row configuration.
4
Verify Active Enrollment Thresholds
Check the total headcount of registered active students listed inside the target batch ledger matrix. Check if the class meets the required minimum quorum of 15 to 20 students to initialize institutional training programs.
5
Configure Calendar Commencement Date
If the active headcount parameter successfully satisfies or surpasses the 15-20 minimum student limit rule, the calendar date input element will unlock. Proceed to choose an official term start date to lock the schedule in.
Validation Constraint Block If the total number of enrollees sits below the absolute minimum requirement of 15 students, the date picker system button remains completely locked and unclickable.
Feature: Attendance Workspace
Attendance Management Module

Tab A: Check & Encode Daily Attendance

1
Access Module Layer
Click the "Attendance Management Module" item link directly within the left sidebar navigation deck.
2
Select Target Training Batch
Inside the active view dashboard, locate the sub-navigation row interface, then click and press the class selection target item to load the corresponding trainee database layout.
3
Configure Session Tracking Date
Choose the exact target calendar index day for which you are taking attendance operations.
4
Modify Trainee Operational Attendance Status Indicators
Select from the dropdown selection arrays matching individual student rows to allocate status values: Present, Absent, or Late.
5
Commit Daily Log Session Metrics
Click the "Save Current Sheet Daily Record" input button to save the dataset configuration into your local relational instance layer.
Feature: Grade Encoding
Encode Competency Marks
1
View Progress and Grading
Access the "Progress and Grading" tab view to display student records alongside structural core performance units.
2
Apply Binary Grading Evaluation Choices
Evaluate laboratory performance and map specific unit dropdown metrics to either Competent (C) or Not Yet Competent (NYC) status markers.
3
Trigger Competency Highlight Action
When a student successfully fulfills the required criteria parameters and hits the pass mark, the system flags them as competent. Upon validation, a dedicated Add Button** icon dynamically appears on the far right edge of that student's row.
4
Compile Batch Competency List
Press the newly rendered Add Button. This action pastes the target student's full name and profile strings directly down into a summary text box block located beneath the grid layout. This block holds the structured list of all competent students from the batch, ready to be quickly copied and pasted into external institutional files.
Feature: Progress Chart of Students
Review Student Progress Charts & Analytics Dashboard
1
Access Real-Time Progress Aggregation Views
Navigate directly to the student tracking summary chart grid panel within your primary grading context view.
2
Evaluate Completion Percentage Indicators

Feature: Laboratory Asset Management
Approve Laboratory Borrower Requests & Logs
1
Load Active Pending Borrower Queues
Navigate to the "Borrower Requests" component display card window to view pending trainee instrumentation and tool demands.
2
Execute Request Determinations (Approve vs. Feedback)
Validate item fields and determine authorization pathways. Instructors can either choose to immediately click Approve to issue the equipment, or click Feedback to handle request blocks.
3
Formulate Operational Feedback Dispatches
If clicking **Feedback**, select or input the specific tracking failure reason. Standard system reasons include: *Target hardware apparatus is not in inventory stock yet*, or *The target borrower has not returned their previous tools*.
Asset Tracking Constraint Approving a request modifies asset status values inside the tool log database, tracking records until the student checks the hardware items back in.
4
Review Integrated Historical Distribution Logs
Scroll directly down below the active requests queue to view the integrated lifecycle logs dashboard layout positioned directly within the module workspace. Confirm clean return timestamps and historical tool intervals here.
Feature: Course Withdrawal Approval
Process Student Course Withdrawals
1
Access Course Withdrawal Processing Matrix
Select the "Withdrawal & Hardships" option block under the sidebar layout index directory mapping to list pending trainee resignation records.
2
Review Listed Student Hardship Justifications
Read through explicit textual reasons submitted by trainees requesting official program exit processing adjustments.
3
Verify Institutional Asset Return Checkboxes
Audit physical inventory obligations by ensuring the specific toggle checkboxes for Student ID Return and Uniform Return are both manually marked off.
Liability Gatekeeper Rule If either the Student ID or Uniform checkboxes remain unchecked, the administrative action engine will throw validation flags and disable the "Approve Program Exit Adjustment" action button.
4
Finalize Academic Drop Terminations
Once liability checkboxes are verified, click the now unlocked "Approve Program Exit Adjustment" confirmation input element to finalize drop procedures and route the termination token to the Registrar Office.
